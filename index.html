Sistema de Base de Datos para una Constructora

Proyecto académico de gestión de información en una empresa constructora

¿De qué trata el proyecto?

Este proyecto consiste en el diseño de un sistema de base de datos para una constructora. El sistema organiza y administra información clave de clientes, trabajadores, planos, proveedores, finanzas, almacén, transporte, seguridad y más. Con él, se busca mejorar la eficiencia operativa y el control de la información en todas las áreas de la empresa.

Problemática

Actualmente, las constructoras suelen enfrentar problemas al manejar la información de manera dispersa: registros duplicados, falta de control en almacén, problemas de comunicación entre áreas y poca seguridad en el almacenamiento de datos. Esto genera retrasos, pérdidas económicas y riesgos legales.

Objetivo

Desarrollar un sistema centralizado de base de datos que permita organizar, consultar y mantener actualizada la información de todos los departamentos de la constructora, mejorando la eficiencia, la seguridad y la toma de decisiones.

Subsistemas

El sistema se divide en los siguientes subsistemas:

  • Clientes
  • Trabajadores
  • Planos
  • Proveedores
  • Finanzas
  • Área Legal
  • Permisos y Licencias
  • Seguros
  • Almacén
  • Transporte
  • Seguridad
  • Ingenieros Especialistas
  • Diseñadores / Paisajistas

Por ahora, solo contienen información las secciones de Almacén, Transporte y Seguridad.
